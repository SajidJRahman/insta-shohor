Frequently Asked Questions

JavaScript is a high-level, interpreted & dynamically-typed programming language. To run JavaScript code & to make it understand & usable by computer, code goes through a compiler program. The program then converts code into bytecode so that computer can read it. On browsers like Google Chrome, JavaScript runs on the V8 high-performance engine & on Mozilla FireFox it runs on SpiderMonkey engine. Despite of JavaScript being an interpreted language, both of the browser's engine uses the Just In Time (JIT) Compiler for faster performance. JIT converts JavaScript code into computer understandable bytecode & runs the code.

On JavaScript, some things runs in synchronous & some things in asynchronous. Some things on JavaScript maintains stack of frames. There are two things to notice, 1: Stack & 2: Queue. All the Stack properties runs first. While Queue properties stay in the queue & if comes first in the queue, runs first. While doing all these, they never block the road of code running, what comes first, runs first. This is what the event loop does. For example, console.log() is synchronous while fetch is asynchronous.

Session storage is temporary & is deleted after browser/website or tab is closed. While local storage is permanent & is not deleted automatically or while closing the browser or website (until deleted manually). Local storage data persists for one month, 6 months, one year or as long as user keeps the data. While session storage data is deleted immediately while closing the browser/website. Session storage data is only available for specific tab, while local storage data is available throughout the browser. For example, shopping websites mostly uses Local Storage to store shopping cart data item.
